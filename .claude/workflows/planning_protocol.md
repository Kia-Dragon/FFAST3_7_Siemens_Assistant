# Planning Output Format

## How Plans Must Be Presented
All plans must be structured as **numbered phases** for clear authorization control.

---

## Plan Structure

```
## Plan: [Brief Title]

### Phase 1: [Phase Name]
- [Specific action items]
- [Files to be modified]

### Phase 2: [Phase Name]
- [Specific action items]
- [Files to be modified]

[...additional phases as needed...]

---
Ready when you say "Make It So" for any phase.
```

---

## Planning Rules
1. **Numbered phases** - Every plan must have numbered phases
2. **Specific actions** - Each phase lists concrete actions to take
3. **Files identified** - List files that will be created/modified
4. **Stop after planning** - After presenting a plan, STOP and WAIT
5. **No execution** - Do not begin work until release phrase is given
6. **Preview only** - Show diffs and commands as text, not executed

---

## After Presenting a Plan
- State "Ready when you say 'Make It So' for [phase/any phase]"
- Do NOT ask "Should I proceed?" or similar
- Do NOT begin execution
- WAIT for the user's explicit release phrase
